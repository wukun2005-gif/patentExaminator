## 快速上手：用 LED 散热案例走一遍完整流程

### 第1步：新建案件

打开 app → 点击「新建案件」→「创建新案件」，填写案件基本信息：

| 表单字段 | 填什么 |
|----------|--------|
| 发明名称 | 一种基于相变材料的LED散热模组 |
| 申请日 | 2024-03-15 |
| 申请号 | 202410567890.1 |
| 申请人 | 深圳光明科技有限公司 |
| 目标权利要求 | 1 |
| 文本版本 | original |

### 第2步：上传申请文件

进入左侧「文档导入」→ 上传 `led-heatsink/申请文件.pdf`

系统自动：读取全文 → 建立段落索引 → 解析权利要求书（提取10条权利要求）

### 第3步：确认文本

检查段落编号是否连续、权利要求是否完整解析。确认后进入下一步。

### 第4步：添加对比文件

进入左侧「文献清单」→ 依次上传3份对比文件：

**对比文件1（D1）**：`led-heatsink/CN108XXXXXXA-散热器.pdf`
- 标题：一种散热器
- 公开号：CN108XXXXXXA
- 公开日：2022-06-15

**对比文件2（D2）**：`led-heatsink/CN109XXXXXXB-热管散热.pdf`
- 标题：一种热管散热装置
- 公开号：CN109XXXXXXB
- 公开日：2022-09-20

**对比文件3（D3）**：`led-heatsink/US20230000XXXA1-热管理.pdf`
- 标题：Thermal Management Device for Semiconductor Light Sources
- 公开号：US2023/0000XXXA1
- 公开日：2023-01-15

### 第5步：文档解读

进入「文档解读」，AI 自动解读申请文件的技术领域、核心方案、主要权利要求。

### 第6步：Claim Chart（特征分解）

进入「Claim Chart」→ 选择权利要求1 → 点击「生成特征分解」。

系统将权利要求1分解为技术特征 A、B、C、D、E，每个特征关联说明书段落。

### 第7步：新颖性分析

进入「新颖性对照」→ 选择对比文件1 → 运行新颖性分析。

系统逐个特征判断是否被对比文件公开，自动标记差异特征。

### 第8步：创造性分析

进入「创造性分析」→ 运行创造性分析。

三步法：最近似现有技术 → 区别特征 → 技术启示判断 → 初步结论。

### 第9步：形式缺陷检测

进入「形式缺陷」→ 运行缺陷检测。扫描权利要求引用关系、说明书支持性等。

### 第10步：导出

进入「导出」→ 选择 HTML 或 Markdown 格式导出审查报告。

## 精简版测试数据（Mini Samples）

`*-mini/` 目录包含复审场景的精简版测试数据，用于开发测试快速迭代，降低 token 消耗。

### 设计原则

**"麻雀虽小，五脏俱全"** — 每个文件控制在 ≤3 页（修改后权利要求书 ≤2 页），但复审流程所需的全部数据元素一个不少。

### 包含的数据元素（16项）

| # | 数据元素 | 所在文件 |
|---|---------|---------|
| 1 | 申请号 | 申请文件.pdf |
| 2 | 申请日 | 申请文件.pdf |
| 3 | 申请人 | 申请文件.pdf |
| 4 | 发明人 | 申请文件.pdf |
| 5 | 发明名称 | 申请文件.pdf |
| 6 | 权利要求书（独立+从属） | 申请文件.pdf |
| 7 | 说明书（技术领域/背景/内容/附图说明） | 申请文件.pdf |
| 8 | 对比文件公开号 | D1/D2 |
| 9 | 对比文件公开日 | D1/D2 |
| 10 | 对比文件权利要求 | D1/D2 |
| 11 | 审查意见通知书（驳回理由） | 第一次审查意见通知书.pdf |
| 12 | 意见陈述书（答辩意见） | 意见陈述书.pdf |
| 13 | 修改后权利要求书 | 修改后权利要求书.pdf |
| 14 | 对比文件技术方案 | D1/D2 |
| 15 | 审查员引用条款 | 第一次审查意见通知书.pdf |
| 16 | 申请人修改标记 | 修改后权利要求书.pdf |

### 文件规格

| 文件类型 | 页数 | 大小 |
|---------|------|------|
| 申请文件 | ≤3页 | ~110KB |
| 对比文件 (D1/D2) | ≤3页 | ~80KB |
| 审查意见通知书 | ≤3页 | ~90KB |
| 意见陈述书 | ≤3页 | ~100KB |
| 修改后权利要求书 | ≤2页 | ~80KB |

### 生成方式

```bash
node scripts/generate-mini-sample-pdfs.js
```

### 与完整版的关系

| | 完整版 (`led-heatsink/`) | 精简版 (`led-heatsink-mini/`) |
|---|---|---|
| 用途 | 完整流程演示、E2E 测试 | 开发测试快速迭代 |
| 申请文件 | 35页 | 3页 |
| 对比文件 | 20页 | 3页 |
| 对比文件数量 | 3个（含英文D3） | 2个（仅中文） |
| 复审文件 | 有 | 有 |
| Token 消耗 | 高 | 低 |
