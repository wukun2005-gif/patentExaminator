# 测试数据样本

本目录包含专利审查全流程所需的测试数据样本。

## 目录结构

```
samples/
├── 01-专利申请文件/
│   └── led-heatsink/申请文件.pdf          # 一种基于相变材料的LED散热模组（30页完整专利申请书）
├── 02-对比文件/
│   ├── CN108XXXXXXA-散热器.txt            # D1：铝合金基板焊接散热器（中文）
│   ├── CN109XXXXXXB-热管散热.txt          # D2：铜-水热管散热装置（中文）
│   └── US20230000XXXA1-热管理.txt         # D3：相变材料热管理装置（英文）
└── README.md
```

## 申请文件说明

`led-heatsink/申请文件.pdf` 是一份完整的中国发明专利申请书（30页），包含：

- 封面（申请号、申请人、发明人等基本信息）
- 权利要求书（10项权利要求，含独立权利要求和从属权利要求）
- 说明书（技术领域、背景技术、发明内容、附图说明、6个实施例、2个对比例、可靠性测试、热设计计算、应用场景）
- 说明书摘要
- 摘要附图说明

该申请书对应 PRD 附录C中的 **G1（Golden Case 1）** 测试用例。

## 快速上手：用 LED 散热案例走一遍完整流程

### 第1步：新建案件

打开 app → 点击「新建案件」→「创建新案件」，填写案件基本信息：

| 表单字段 | 填什么 |
|----------|--------|
| 发明名称 | 一种基于相变材料的LED散热模组 |
| 申请日 | 2024-03-15 |
| 申请号 | 202410567890.1 |
| 申请人 | 深圳光明科技有限公司 |
| 目标权利要求 | 1 |
| 文本版本 | original |

### 第2步：上传申请文件

进入左侧「文档导入」→ 上传 `01-专利申请文件/led-heatsink/申请文件.pdf`

系统自动：读取全文 → 建立段落索引 → 解析权利要求书（提取10条权利要求）

### 第3步：确认文本

检查段落编号是否连续、权利要求是否完整解析。确认后进入下一步。

### 第4步：添加对比文件

进入左侧「文献清单」→ 依次上传3份对比文件：

**对比文件1（D1）**：`02-对比文件/CN108XXXXXXA-散热器.txt`
- 标题：一种散热器
- 公开号：CN108XXXXXXA
- 公开日：2022-06-15

**对比文件2（D2）**：`02-对比文件/CN109XXXXXXB-热管散热.txt`
- 标题：一种热管散热装置
- 公开号：CN109XXXXXXB
- 公开日：2022-09-20

**对比文件3（D3）**：`02-对比文件/US20230000XXXA1-热管理.txt`
- 标题：Thermal Management Device for Semiconductor Light Sources
- 公开号：US2023/0000XXXA1
- 公开日：2023-01-15

### 第5步：文档解读

进入「文档解读」，AI 自动解读申请文件的技术领域、核心方案、主要权利要求。

### 第6步：Claim Chart（特征分解）

进入「Claim Chart」→ 选择权利要求1 → 点击「生成特征分解」。

系统将权利要求1分解为技术特征 A、B、C、D、E，每个特征关联说明书段落。

### 第7步：新颖性分析

进入「新颖性对照」→ 选择对比文件1 → 运行新颖性分析。

系统逐个特征判断是否被对比文件公开，自动标记差异特征。

### 第8步：创造性分析

进入「创造性分析」→ 运行创造性分析。

三步法：最近似现有技术 → 区别特征 → 技术启示判断 → 初步结论。

### 第9步：形式缺陷检测

进入「形式缺陷」→ 运行缺陷检测。扫描权利要求引用关系、说明书支持性等。

### 第10步：导出

进入「导出」→ 选择 HTML 或 Markdown 格式导出审查报告。
