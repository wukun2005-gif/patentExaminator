# 测试数据样本

本目录包含专利审查全流程所需的各类测试数据样本。

## 专利申请文件 = 一个完整文档

实际专利提交中，申请人提交的是一个完整文件，包含：

```
【发明名称】
【技术领域】
【背景技术】
【发明内容】
【具体实施方式】
【权利要求书】    ← 系统自动从这里解析出权利要求
【摘要】
```

权利要求书不是单独提交的文件，而是申请文件末尾的一个章节。本目录中每个「申请文件.txt」都是这样一个完整文档。系统会自动从中提取"权利要求书"章节并解析为独立的权利要求条目。

## 快速上手：用 LED 散热案例走一遍完整流程

### 第1步：新建案件

打开 app → 点击「新建案件」→「创建新案件」，填写案件基本信息：

| 表单字段 | 填什么 |
|----------|--------|
| 发明名称 | 一种基于相变材料的LED散热模组 |
| 申请日 | 2024-03-15 |
| 申请号 | 202410567890.1 |
| 申请人 | 深圳光明科技有限公司 |
| 目标权利要求 | 1 |
| 文本版本 | original |

### 第2步：上传申请文件

进入左侧「文档导入」→ 上传 `01-专利申请文件/led-heatsink/申请文件.txt`

系统自动：读取全文 → 建立段落索引 → 解析权利要求书（提取7条权利要求）

### 第3步：确认文本

检查段落编号是否连续、权利要求是否完整解析。确认后进入下一步。

### 第4步：添加对比文件

进入左侧「文献清单」→ 上传对比文件：

**对比文件1**：`02-对比文件/CN108XXXXXXA-散热器.txt`
- 标题：一种散热器
- 公开号：CN108XXXXXXA
- 公开日：2022-06-15

**对比文件2**：`02-对比文件/US20230000XXXA1-热管理.txt`
- 标题：Thermal Management Device for Semiconductor Light Sources
- 公开号：US2023/0000XXXA1
- 公开日：2023-01-15

### 第5步：文档解读

进入「文档解读」，AI 自动解读申请文件的技术领域、核心方案、主要权利要求。

### 第6步：Claim Chart（特征分解）

进入「Claim Chart」→ 选择权利要求1 → 点击「生成特征分解」。

系统将权利要求1分解为技术特征 A、B、C、D、E，每个特征关联说明书段落。

### 第7步：新颖性分析

进入「新颖性对照」→ 选择对比文件1 → 运行新颖性分析。

系统逐个特征判断是否被对比文件公开，自动标记差异特征。

### 第8步：创造性分析

进入「创造性分析」→ 运行创造性分析。

三步法：最近似现有技术 → 区别特征 → 技术启示判断 → 初步结论。

### 第9步：形式缺陷检测

进入「形式缺陷」→ 运行缺陷检测。扫描权利要求引用关系、说明书支持性等。

### 第10步：导出

进入「导出」→ 选择 HTML 或 Markdown 格式导出审查报告。

## 在 app 中使用引导

app 顶部导航栏有「引导」按钮，点击后提供两种模式：

- **快速体验**：一键加载预置案例，所有数据已填充，直接浏览各功能页面
- **Sample 引导**：按步骤引导你用 sample 文件走完整流程，每一步告诉你做什么、用哪个文件、填什么字段

## 与 fixture 的关系

- `shared/src/fixtures/*.json` — 结构化数据，可直接「加载预置案例」一步到位
- `samples/` — 原始文档，需要走完整流程（上传→解析→分析），适合测试真实工作流
